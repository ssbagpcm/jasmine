# Frontend

Use this skill for web pages, application screens, component styling, responsive layouts, and visual polish.

## Workflow

1. Inspect the existing framework, routes, design tokens, and component patterns.
2. Identify the exact user-visible surfaces affected by the request.
3. Reuse the project stack and existing components before adding dependencies.
4. Implement a coherent visual hierarchy with clear spacing, readable typography, and responsive states.
5. Run the narrow build or test command first.
6. Start the application when possible and inspect the real page at desktop and mobile widths.
7. Fix visible regressions, overflow, alignment issues, and console errors before finishing.

## Quality Bar

- Avoid generic placeholder layouts when the request calls for design work.
- Preserve accessibility: semantic controls, keyboard usability, labels, and sufficient contrast.
- Check loading, empty, error, hover, focus, and disabled states when relevant.
- Keep changes scoped to the requested interface.
